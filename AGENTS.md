## Project Overview

This is a personal academic website for Dr. Yujun Cai (Lecturer at the University of Queensland) built with Jekyll and hosted on GitHub Pages. The website showcases academic profile, publications, teaching, and services.

**Live Site**: https://vanoracai.github.io/

## Technology Stack

- **Static Site Generator**: Jekyll (GitHub Pages compatible)
- **Theme**: Custom theme based on Jekyll homepage layout
- **Language**: Ruby (Jekyll), Python (bibliography processing)
- **Hosting**: GitHub Pages

## Development Commands

### Local Development

```bash
# Install dependencies
bundle install

# Serve the site locally (with live reload)
bundle exec jekyll serve

# The site will be available at http://localhost:4000
```

### Build Site

```bash
# Build the static site (output to _site/)
bundle exec jekyll build
```

### Deploy

The site automatically deploys to GitHub Pages when changes are pushed to the main branch.

## Bibliography Management

The website includes Python scripts to manage academic publications from BibTeX files:

### Generate HTML Publications List

```bash
# Convert citations.bib to HTML format for the website
python bib_to_html.py

# This generates _includes/output_file.md which is included in publications.md
```

### Generate LaTeX CV Publications List

```bash
# Convert citations.bib to LaTeX format for CV
python bib_to_latex.py

# This generates paper_list.tex for inclusion in CV documents
```

### Fill BibTeX URLs

```bash
# Automatically populate missing URLs in citations.bib
python fill_bib_url.py
```

**Workflow**: When adding new publications:
1. Update `citations.bib` with new entries
2. Run `python bib_to_html.py` to generate the HTML version
3. Run `python bib_to_latex.py` to update the LaTeX CV list
4. Commit both the .bib file and generated files

## Architecture Notes

### Publication System

The publication management system uses a three-stage approach:
1. **Source**: `citations.bib` - Single source of truth for all publications in BibTeX format
2. **Processing**: Python scripts parse and format the BibTeX data
3. **Output**:
   - HTML format for website (`_includes/output_file.md`)
   - LaTeX format for CV (`paper_list.tex`)

The `bib_to_html.py` script includes venue abbreviation mappings (e.g., "Conference on Computer Vision and Pattern Recognition" → "CVPR") to display conference names consistently.

### Jekyll Configuration

The site uses:
- Jekyll's `github-pages` gem for compatibility
- `webrick` gem (required for Ruby 3.0+)
- Custom `enable_hopkins_logo` and `enable_last_update` flags in config

### Content Organization

- Main pages are markdown files in the root directory
- Content is modularized using Jekyll's `{% include %}` system
- Dynamic content (publications) is generated from structured data (BibTeX)

## Common Tasks

### Update Personal Information

Edit `_config.yml` to change:
- Name, position, affiliation
- Email, links (Google Scholar, LinkedIn, CV)
- Profile photo paths

### Add News Item

Edit `index.md` in the `## News` section. News items are manually maintained in reverse chronological order.

### Update CV

1. Replace `yujun_cv.pdf` with the new version
2. Ensure the filename matches the `cv_link` setting in `_config.yml`

### Modify Styling

- Edit files in `_sass/` directory for SASS styles
- Or modify `assets/` directory for additional CSS/JS

### Test Before Deploy

Always run `bundle exec jekyll serve` and check http://localhost:4000 before pushing changes to ensure:
- Pages render correctly
- Links work
- No Jekyll build errors
