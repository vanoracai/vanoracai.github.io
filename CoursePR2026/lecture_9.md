## Notation and Variable Definitions

This chapter introduces several symbols that look similar but play different roles. The most important distinction is between an **observed data point** and its **unknown cluster identity**.

| Symbol | Definition |
|--------|------------|
| $\mathbf{x}_n\in\mathbb{R}^D$ | The $n$th observed data point. |
| $N$ | Number of data points. |
| $D$ | Dimensionality of each data point. |
| $K$ | Number of clusters or mixture components. It is assumed known during basic K-means and GMM training. |
| $\boldsymbol{\mu}_k$ | Centre of cluster $k$, or mean of Gaussian component $k$. |
| $r_{nk}\in\{0,1\}$ | Hard K-means assignment. It equals 1 when point $n$ belongs to cluster $k$. |
| $\mathbf{z}_n$ | A hidden 1-of-$K$ variable indicating which mixture component generated $\mathbf{x}_n$. |
| $\pi_k$ | Mixing coefficient of component $k$, with $\pi_k\geq 0$ and $\sum_k\pi_k=1$. |
| $\boldsymbol{\Sigma}_k$ | Covariance matrix of Gaussian component $k$. |
| $\gamma(z_{nk})$ | Responsibility: posterior probability that component $k$ generated point $n$. |
| $N_k$ | Effective number of data points assigned to component $k$: $N_k=\sum_n\gamma(z_{nk})$. |
| $J$ | K-means distortion objective. |
| $\mathcal{N}(\mathbf{x}\mid\boldsymbol{\mu},\boldsymbol{\Sigma})$ | Multivariate Gaussian density. |
| $\boldsymbol{\theta}$ | Generic notation for all model parameters. |

> **Teaching focus.** The two assignment variables are different:
>
> - $r_{nk}$ is a **hard decision**: either 0 or 1.
> - $\gamma(z_{nk})$ is a **soft probability** between 0 and 1.
>
> This hard-versus-soft distinction is the main conceptual bridge from K-means to Gaussian mixture models.

# §0 Learning Viewpoint and Chapter Roadmap

> 📖 Textbook Ch. 9 opening; §9.1–§9.4

## 0.1 What This Chapter Is Really About

This chapter answers a common unsupervised-learning question:

> We observe many data points, but nobody tells us which point belongs to which group. How can a model discover useful groups by itself?

There are two levels of answer.

| Method | Main Idea | Output |
|--------|-----------|--------|
| **K-means** | Assign every point to its nearest centre and repeatedly update the centres. | One hard cluster label per point. |
| **Gaussian mixture model (GMM)** | Assume the data were generated by several Gaussian components and infer the probability of each possible component. | A probability distribution over cluster identities. |

The expectation-maximization algorithm connects these ideas. EM is useful when a model would be easy to learn **if hidden information were known**, but that hidden information is not observed.

The central pattern is

$$
\text{guess hidden assignments}
\longrightarrow
\text{update model parameters}
\longrightarrow
\text{guess again}.
$$

This pattern appears far beyond Gaussian mixtures.

## 0.2 Why Mixture Models Are Useful

A single Gaussian describes one approximately elliptical cloud of data. Real data often contain several groups, operating modes, populations, or signal regimes.

Examples include:

- measurements from several types of devices,
- background noise plus several signal classes,
- customer groups with different behaviour,
- multiple operating states of an electrical system,
- colour clusters in an image,
- acoustic features from different speakers or phonetic units.

A mixture model builds a complicated distribution from simple components:

$$
\text{complex data distribution}
=
\text{weighted combination of simple distributions}.
$$

## 0.3 Learning Goals

After this chapter, students should be able to:

1. explain the two alternating steps of K-means;
2. derive the K-means centre update without skipping steps;
3. write the probability density of a Gaussian mixture;
4. explain a latent component variable $\mathbf{z}$;
5. compute a responsibility using Bayes' theorem;
6. explain the E-step and M-step in plain language;
7. use the EM update equations for means, covariances, and mixing coefficients;
8. compare hard K-means assignments with soft GMM assignments;
9. identify practical problems such as bad initialization, local optima, and covariance collapse.

## 0.4 Scope Decisions for This Course

The textbook contains a broad and elegant treatment of EM. For this EE undergraduate/master's course, we keep the parts that are most useful for understanding and implementation.

### Covered carefully

- K-means and its objective;
- image segmentation and vector quantization;
- Gaussian mixtures and latent component identities;
- responsibilities;
- EM for Gaussian mixtures;
- practical initialization and failure modes;
- the conceptual relation between K-means and EM.

### Introduced only briefly

- complete versus incomplete data;
- the general EM template;
- why the likelihood does not decrease after a correct EM cycle;
- model selection for the number of components.

### Deliberately not derived in detail

- online K-means through Robbins–Monro theory;
- Bernoulli-mixture EM equations;
- EM for Bayesian linear regression and relevance vector machines;
- the full KL-divergence/ELBO proof of general EM;
- generalized EM and incremental EM theory.

These topics are mathematically useful, but they are not necessary for learning the main algorithm and would make this lecture less focused.

## 0.5 Chapter Roadmap

The learning path is:

$$
\boxed{\text{K-means}}
\rightarrow
\boxed{\text{hard assignment}}
\rightarrow
\boxed{\text{Gaussian mixture}}
\rightarrow
\boxed{\text{hidden component}}
\rightarrow
\boxed{\text{soft responsibility}}
\rightarrow
\boxed{\text{EM}}.
$$

A useful interpretation is:

- K-means asks, “Which centre is nearest?”
- A GMM asks, “How probable is each possible component?”
- EM alternates between answering that probability question and updating the component parameters.

# §1 K-means Clustering

> 📖 Textbook §9.1, pp. 424–430

## 1.1 The Clustering Problem

Suppose we observe

$$
\mathcal{D}=\{\mathbf{x}_1,\mathbf{x}_2,\ldots,\mathbf{x}_N\},
\qquad
\mathbf{x}_n\in\mathbb{R}^D.
$$

There are no target labels. We want to divide these data into $K$ groups.

For each cluster, introduce a centre

$$
\boldsymbol{\mu}_k\in\mathbb{R}^D,
\qquad k=1,\ldots,K.
$$

The intuitive goal is simple:

> Data points in the same cluster should be close to the same centre.

K-means uses squared Euclidean distance:

$$
\left\|\mathbf{x}_n-\boldsymbol{\mu}_k\right\|^2.
$$

For a two-dimensional vector,

$$
\mathbf{x}_n=
\begin{bmatrix}
x_{n1}\\
x_{n2}
\end{bmatrix},
\qquad
\boldsymbol{\mu}_k=
\begin{bmatrix}
\mu_{k1}\\
\mu_{k2}
\end{bmatrix},
$$

so

$$
\left\|\mathbf{x}_n-\boldsymbol{\mu}_k\right\|^2
=
(x_{n1}-\mu_{k1})^2+(x_{n2}-\mu_{k2})^2.
$$

No square root is required. The nearest centre under squared distance is the same as the nearest centre under ordinary Euclidean distance.

## 1.2 Hard Assignment Variables

For every point $n$ and cluster $k$, define

$$
r_{nk}\in\{0,1\}.
$$

The meaning is

$$
r_{nk}=
\begin{cases}
1, & \text{if point }n\text{ is assigned to cluster }k,\\
0, & \text{otherwise}.
\end{cases}
$$

Each point must belong to exactly one cluster:

$$
\sum_{k=1}^K r_{nk}=1.
$$

For example, with $K=3$,

$$
(r_{n1},r_{n2},r_{n3})=(0,1,0)
$$

means that point $n$ belongs to cluster 2.

## 1.3 The K-means Objective

The distortion objective is

$$
J
=
\sum_{n=1}^{N}
\sum_{k=1}^{K}
r_{nk}
\left\|\mathbf{x}_n-\boldsymbol{\mu}_k\right\|^2.
$$

Read the formula from inside to outside:

1. compute the squared distance from point $n$ to centre $k$;
2. multiply by $r_{nk}$;
3. only the assigned cluster has $r_{nk}=1$;
4. add the assigned distances over all points.

Thus $J$ is simply the total within-cluster squared distance.

### Small interpretation example

Suppose a point has distances

$$
\|\mathbf{x}_n-\boldsymbol{\mu}_1\|^2=3,
\qquad
\|\mathbf{x}_n-\boldsymbol{\mu}_2\|^2=10.
$$

If the point is assigned to cluster 1, then

$$
(r_{n1},r_{n2})=(1,0),
$$

and its contribution to $J$ is

$$
1\cdot 3+0\cdot 10=3.
$$

## 1.4 K-means Alternates Between Two Easy Problems

Trying to optimize all assignments and all centres simultaneously is difficult. K-means instead alternates between two simpler steps.

### Assignment step

Keep the centres fixed. Assign each point to its nearest centre:

$$
r_{nk}=
\begin{cases}
1,
& k=\displaystyle\arg\min_j
\|\mathbf{x}_n-\boldsymbol{\mu}_j\|^2,\\
0, & \text{otherwise}.
\end{cases}
$$

### Centre-update step

Keep the assignments fixed. Replace every centre by the mean of the points assigned to it.

The two operations are repeated until the assignments or the objective stop changing.

> **Terminology note.** Bishop calls these two K-means operations an E-step and an M-step to emphasize their relationship with EM. In many modern courses and software libraries, they are simply called the assignment step and update step. For K-means, these names are easier and less confusing.

## 1.5 Deriving the Centre Update Step by Step

For one particular cluster $k$, the part of the objective involving $\boldsymbol{\mu}_k$ is

$$
J_k
=
\sum_{n=1}^{N}
r_{nk}
\left\|\mathbf{x}_n-\boldsymbol{\mu}_k\right\|^2.
$$

To minimize $J_k$, differentiate with respect to $\boldsymbol{\mu}_k$.

First use

$$
\frac{\partial}{\partial\boldsymbol{\mu}_k}
\left\|\mathbf{x}_n-\boldsymbol{\mu}_k\right\|^2
=
2(\boldsymbol{\mu}_k-\mathbf{x}_n).
$$

Therefore

$$
\frac{\partial J_k}{\partial\boldsymbol{\mu}_k}
=
2\sum_{n=1}^{N}r_{nk}(\boldsymbol{\mu}_k-\mathbf{x}_n).
$$

Set the derivative to zero:

$$
2\sum_{n=1}^{N}r_{nk}(\boldsymbol{\mu}_k-\mathbf{x}_n)=0.
$$

Divide by 2:

$$
\sum_{n=1}^{N}r_{nk}\boldsymbol{\mu}_k
-
\sum_{n=1}^{N}r_{nk}\mathbf{x}_n
=0.
$$

Because $\boldsymbol{\mu}_k$ does not depend on $n$,

$$
\boldsymbol{\mu}_k
\sum_{n=1}^{N}r_{nk}
=
\sum_{n=1}^{N}r_{nk}\mathbf{x}_n.
$$

Finally,

$$
\boxed{
\boldsymbol{\mu}_k
=
\frac{\sum_{n=1}^{N}r_{nk}\mathbf{x}_n}
{\sum_{n=1}^{N}r_{nk}}
}.
$$

The denominator counts how many points are assigned to cluster $k$. The numerator adds those points. Therefore the result is exactly their sample mean.

## 1.6 A Small One-Dimensional Worked Example

Consider the data

$$
\{1,2,3,8,9,10\}
$$

and choose $K=2$.

Initialize the centres as

$$
\mu_1^{(0)}=2,
\qquad
\mu_2^{(0)}=8.
$$

### Assignment step

For each point, compare its distance to the two centres.

| Point $x_n$ | Distance to 2 | Distance to 8 | Assignment |
|-------------|---------------|---------------|------------|
| 1 | 1 | 7 | Cluster 1 |
| 2 | 0 | 6 | Cluster 1 |
| 3 | 1 | 5 | Cluster 1 |
| 8 | 6 | 0 | Cluster 2 |
| 9 | 7 | 1 | Cluster 2 |
| 10 | 8 | 2 | Cluster 2 |

### Centre-update step

The new centres are

$$
\mu_1^{(1)}=\frac{1+2+3}{3}=2,
$$

and

$$
\mu_2^{(1)}=\frac{8+9+10}{3}=9.
$$

Run the assignment step again. The assignments remain unchanged, so the algorithm has converged.

The example is intentionally simple. Its purpose is to show that K-means is not mysterious:

> assign to the nearest centre, then recompute the average.

## 1.7 The Textbook's Old Faithful Example

> ![Figure 9.1](./CoursePR2026/Fig/Chapter_9/lecture_fig_9_1__textbook_fig_9_1__p426.png)
>
> *Figure 9.1 (Textbook Fig. 9.1, p. 426): Successive K-means assignment and centre-update steps on the standardized Old Faithful data. Crosses denote cluster centres. The magenta lines are boundaries between nearest-centre regions.*

The figure should be read row by row.

- In panel (a), the centres are poorly initialized.
- In panel (b), every point receives a hard red or blue assignment.
- In panel (c), each centre moves to the mean of its assigned points.
- The same two operations repeat in panels (d)–(i).

The final result is sensible even though the initial centres were poor. However, this does not mean every initialization gives the same answer.

## 1.8 Why the Objective Decreases

During the assignment step, each point moves to the nearest available centre. This cannot increase its squared distance.

During the centre-update step, the mean is the point that minimizes the sum of squared distances for the current cluster membership. This also cannot increase $J$.

Therefore every full K-means iteration satisfies

$$
J^{(t+1)}\leq J^{(t)}.
$$

> ![Figure 9.2](./CoursePR2026/Fig/Chapter_9/lecture_fig_9_2__textbook_fig_9_2__p427.png)
>
> *Figure 9.2 (Textbook Fig. 9.2, p. 427): The K-means objective decreases after each assignment step and centre-update step, then becomes constant at convergence.*

This guarantees convergence to a stable solution after a finite number of hard-assignment configurations. It does **not** guarantee the globally best clustering.

## 1.9 Local Minima and Initialization

K-means can converge to different solutions from different initial centres.

A poor initialization can produce:

- a bad local minimum;
- an empty cluster;
- several centres placed in one dense region;
- slow convergence.

Useful practices include:

1. run K-means several times with different random seeds;
2. retain the solution with the smallest final $J$;
3. use K-means++ style initialization so that initial centres are spread apart;
4. standardize features when their numerical scales are very different.

### Why feature scaling matters

Suppose feature 1 ranges from 0 to 1, while feature 2 ranges from 0 to 10,000. Euclidean distance will be dominated by feature 2. Standardization often uses

$$
\widetilde{x}_{nd}
=
\frac{x_{nd}-\text{mean}_d}{\text{standard deviation}_d}.
$$

Scaling is not always required, but it should be considered whenever feature units differ.

## 1.10 Image Segmentation and Vector Quantization

K-means can cluster image pixels using their colour vectors

$$
\mathbf{x}_n=(R_n,G_n,B_n)^T.
$$

Every cluster centre becomes a representative colour. Each original pixel is replaced by the colour of its nearest centre.

> ![Figure 9.3](./CoursePR2026/Fig/Chapter_9/lecture_fig_9_3__textbook_fig_9_3__p429.png)
>
> *Figure 9.3 (Textbook Fig. 9.3, p. 429): K-means image segmentation and colour quantization. Increasing $K$ preserves more colours and improves visual quality, but reduces compression.*

For $K=2$, the image is represented by only two code-book colours. For $K=10$, more visual detail is retained.

This is an example of **lossy compression**:

- store $K$ representative colour vectors;
- for each pixel, store only the index of its nearest representative;
- reconstruct each pixel using its centre colour.

If an index requires approximately $\log_2 K$ bits, then a small $K$ gives strong compression but poor reconstruction.

## 1.11 Strengths and Limitations of K-means

| Strength | Limitation |
|----------|------------|
| Easy to understand and implement | Requires the number of clusters $K$ in advance |
| Usually fast | Sensitive to initialization |
| Scales to large data sets | Uses only squared Euclidean distance |
| Good baseline for clustering | Produces hard assignments only |
| Useful for vector quantization | Prefers roughly spherical, similarly sized clusters |
| Often useful for initializing GMMs | Does not provide a probability density or uncertainty |

The last two limitations motivate Gaussian mixture models.

# §2 Gaussian Mixture Models

> 📖 Textbook §9.2, pp. 430–438

## 2.1 Why a Single Gaussian Is Often Not Enough

A multivariate Gaussian can model one approximately elliptical cloud. Its parameters are

$$
\boldsymbol{\mu}
\quad\text{and}\quad
\boldsymbol{\Sigma}.
$$

The mean determines the centre. The covariance determines the size, orientation, and shape of the ellipse.

However, one Gaussian cannot represent several separated groups well. If data contain two distant clusters, fitting one Gaussian often places its mean between them, in a low-density region where few observations actually occur.

A Gaussian mixture solves this by using several Gaussian components.

## 2.2 The Gaussian Mixture Density

A mixture of $K$ Gaussian components is

$$
\boxed{
p(\mathbf{x})
=
\sum_{k=1}^{K}
\pi_k
\mathcal{N}(\mathbf{x}\mid\boldsymbol{\mu}_k,\boldsymbol{\Sigma}_k)
}.
$$

Each component has three kinds of parameters:

| Parameter | Meaning |
|-----------|---------|
| $\pi_k$ | How common component $k$ is before seeing $\mathbf{x}$ |
| $\boldsymbol{\mu}_k$ | Centre of component $k$ |
| $\boldsymbol{\Sigma}_k$ | Shape and spread of component $k$ |

The mixing coefficients must satisfy

$$
\pi_k\geq 0,
\qquad
\sum_{k=1}^{K}\pi_k=1.
$$

These conditions make $\{\pi_k\}$ a valid probability distribution over components.

### A one-dimensional example

Suppose

$$
\pi_1=0.7,
\qquad
\pi_2=0.3.
$$

Then

$$
p(x)
=
0.7\,\mathcal{N}(x\mid\mu_1,\sigma_1^2)
+
0.3\,\mathcal{N}(x\mid\mu_2,\sigma_2^2).
$$

Before observing $x$, component 1 is more likely to generate a sample. But after observing a particular $x$, component 2 could still be more plausible if $x$ lies close to $\mu_2$.

This distinction between **prior component probability** and **posterior component probability** is essential.

## 2.3 A Generative Story

The mixture equation becomes easier to understand as a two-stage sampling process.

1. Choose a component identity $z$ according to the probabilities $\pi_1,\ldots,\pi_K$.
2. Generate $\mathbf{x}$ from that component's Gaussian distribution.

For example, with three components:

$$
z\sim\operatorname{Categorical}(\pi_1,\pi_2,\pi_3),
$$

then

$$
\mathbf{x}\mid z=k
\sim
\mathcal{N}(\boldsymbol{\mu}_k,\boldsymbol{\Sigma}_k).
$$

The component identity is not usually recorded in the observed data. It is therefore a **latent variable（隐变量）**.

## 2.4 The Latent Variable $\mathbf{z}$

Bishop represents the component identity using a 1-of-$K$ binary vector

$$
\mathbf{z}=(z_1,z_2,\ldots,z_K)^T,
$$

with

$$
z_k\in\{0,1\},
\qquad
\sum_{k=1}^K z_k=1.
$$

For $K=3$:

$$
\mathbf{z}=(0,1,0)^T
$$

means that the second component generated the point.

> ![Figure 9.4](./CoursePR2026/Fig/Chapter_9/lecture_fig_9_4__textbook_fig_9_4__p431.png)
>
> *Figure 9.4 (Textbook Fig. 9.4, p. 431): A simple directed graph for a mixture model. The hidden component identity $z$ determines the distribution from which the observed variable $x$ is generated.*

The graphical-model factorization is

$$
p(\mathbf{x},\mathbf{z})
=
p(\mathbf{z})p(\mathbf{x}\mid\mathbf{z}).
$$

## 2.5 Recovering the Mixture by Marginalization

Suppose $z$ can take $K$ possible component values. The observed density is obtained by summing over the unknown component:

$$
p(\mathbf{x})
=
\sum_{\mathbf{z}}
p(\mathbf{x},\mathbf{z}).
$$

Use the product rule:

$$
p(\mathbf{x})
=
\sum_{\mathbf{z}}
p(\mathbf{z})p(\mathbf{x}\mid\mathbf{z}).
$$

For component $k$,

$$
p(z_k=1)=\pi_k,
$$

and

$$
p(\mathbf{x}\mid z_k=1)
=
\mathcal{N}(\mathbf{x}\mid\boldsymbol{\mu}_k,\boldsymbol{\Sigma}_k).
$$

Therefore

$$
p(\mathbf{x})
=
\sum_{k=1}^{K}
\pi_k
\mathcal{N}(\mathbf{x}\mid\boldsymbol{\mu}_k,\boldsymbol{\Sigma}_k).
$$

This derivation shows a general latent-variable idea:

> A complicated observed distribution can be obtained by marginalizing a simpler joint distribution over observed and hidden variables.

## 2.6 Responsibilities: Soft Cluster Membership

After observing a point $\mathbf{x}$, we want the probability that component $k$ generated it:

$$
p(z_k=1\mid\mathbf{x}).
$$

This posterior probability is called the **responsibility** of component $k$ for explaining $\mathbf{x}$:

$$
\gamma(z_k)
\equiv
p(z_k=1\mid\mathbf{x}).
$$

Apply Bayes' theorem:

$$
p(z_k=1\mid\mathbf{x})
=
\frac{p(z_k=1)p(\mathbf{x}\mid z_k=1)}{p(\mathbf{x})}.
$$

Substitute the mixture quantities:

$$
p(z_k=1)=\pi_k,
$$

$$
p(\mathbf{x}\mid z_k=1)
=
\mathcal{N}(\mathbf{x}\mid\boldsymbol{\mu}_k,\boldsymbol{\Sigma}_k),
$$

and

$$
p(\mathbf{x})
=
\sum_{j=1}^{K}
\pi_j
\mathcal{N}(\mathbf{x}\mid\boldsymbol{\mu}_j,\boldsymbol{\Sigma}_j).
$$

Therefore

$$
\boxed{
\gamma(z_k)
=
\frac{
\pi_k\mathcal{N}(\mathbf{x}\mid\boldsymbol{\mu}_k,\boldsymbol{\Sigma}_k)
}{
\sum_{j=1}^{K}
\pi_j\mathcal{N}(\mathbf{x}\mid\boldsymbol{\mu}_j,\boldsymbol{\Sigma}_j)
}
}.
$$

For data point $n$, write

$$
\gamma(z_{nk})
=
p(z_{nk}=1\mid\mathbf{x}_n).
$$

Responsibilities satisfy

$$
0\leq\gamma(z_{nk})\leq 1,
$$

and for each point,

$$
\sum_{k=1}^{K}\gamma(z_{nk})=1.
$$

## 2.7 A Numerical Responsibility Example

Suppose there are two components. For one observed point $x$:

$$
\pi_1=0.6,
\qquad
\pi_2=0.4.
$$

Assume the component likelihood values at this point are

$$
\mathcal{N}(x\mid\mu_1,\sigma_1^2)=0.2,
$$

and

$$
\mathcal{N}(x\mid\mu_2,\sigma_2^2)=0.5.
$$

First compute the unnormalized component scores:

$$
\pi_1\mathcal{N}_1(x)
=0.6\times 0.2=0.12,
$$

$$
\pi_2\mathcal{N}_2(x)
=0.4\times 0.5=0.20.
$$

The evidence is

$$
p(x)=0.12+0.20=0.32.
$$

Therefore

$$
\gamma(z_1)=\frac{0.12}{0.32}=0.375,
$$

and

$$
\gamma(z_2)=\frac{0.20}{0.32}=0.625.
$$

Although component 1 had the larger prior probability, the observed point is more compatible with component 2. The posterior responsibility correctly combines both pieces of information.

## 2.8 Complete Data, Incomplete Data, and Soft Colouring

> ![Figure 9.5](./CoursePR2026/Fig/Chapter_9/lecture_fig_9_5__textbook_fig_9_5__p433.png)
>
> *Figure 9.5 (Textbook Fig. 9.5, p. 433): (a) Samples shown with their true generating components, (b) the same observations after component labels are hidden, and (c) the same observations coloured according to posterior responsibilities.*

The panels illustrate three different information settings.

### Panel (a): complete data

Both $\mathbf{x}_n$ and its component label $\mathbf{z}_n$ are known.

Learning is easy: fit one Gaussian to each known group.

### Panel (b): incomplete data

Only $\mathbf{x}_n$ is observed. The component identity is hidden.

This is the actual mixture-model learning problem.

### Panel (c): posterior responsibilities

Each point receives a mixture of colours. A point close to an overlap region has uncertain membership and is shown using several colours.

This is a useful visual interpretation:

> responsibility is a soft colour mixture describing how strongly each component explains the point.

## 2.9 GMM as Density Estimation and Clustering

A Gaussian mixture can be used in two related ways.

### Density estimation

The model estimates

$$
p(\mathbf{x}).
$$

This supports tasks such as anomaly detection, data generation, likelihood evaluation, and probabilistic inference.

### Clustering

A hard cluster label can be produced after fitting by choosing

$$
\widehat{k}_n
=
\arg\max_k\gamma(z_{nk}).
$$

But the full responsibility vector is often more informative than the hard label.

For example,

$$
(0.99,0.01)
$$

and

$$
(0.51,0.49)
$$

produce the same hard label, but they represent very different levels of confidence.

## 2.10 Covariance Choices

A full covariance matrix gives every component a flexible ellipse. It also requires many parameters.

| Covariance type | Shape | Parameters per component | Typical use |
|-----------------|-------|--------------------------|-------------|
| Spherical | Circle/sphere | 1 variance | Fast, simple baseline |
| Diagonal | Axis-aligned ellipse | $D$ variances | High-dimensional data with limited samples |
| Tied/shared | Same covariance for all components | One covariance shared | Fewer parameters, common shape |
| Full | Arbitrarily oriented ellipse | $D(D+1)/2$ | Flexible when enough data are available |

For EE data with many dimensions but limited samples, diagonal covariance is often a safer starting point than full covariance.

## 2.11 What the Number of Components Means

The number of mixture components $K$ is a model choice. It does not always equal the number of meaningful real-world classes.

One physical class may require several Gaussian components if its distribution is non-Gaussian. Conversely, two semantic classes may overlap so strongly that an unsupervised model does not separate them.

Therefore:

> mixture component is a statistical concept, not automatically a semantic label.

# §3 Why Direct Maximum Likelihood Is Difficult

> 📖 Textbook §9.2.1, pp. 432–435

## 3.1 The GMM Log Likelihood

For independent observations

$$
\mathbf{X}=\{\mathbf{x}_1,\ldots,\mathbf{x}_N\},
$$

the likelihood is

$$
p(\mathbf{X}\mid\boldsymbol{\pi},\boldsymbol{\mu},\boldsymbol{\Sigma})
=
\prod_{n=1}^{N}
\left[
\sum_{k=1}^{K}
\pi_k
\mathcal{N}(\mathbf{x}_n\mid\boldsymbol{\mu}_k,\boldsymbol{\Sigma}_k)
\right].
$$

Taking the logarithm gives

$$
\boxed{
\ln p(\mathbf{X}\mid\boldsymbol{\pi},\boldsymbol{\mu},\boldsymbol{\Sigma})
=
\sum_{n=1}^{N}
\ln
\left[
\sum_{k=1}^{K}
\pi_k
\mathcal{N}(\mathbf{x}_n\mid\boldsymbol{\mu}_k,\boldsymbol{\Sigma}_k)
\right]
}.
$$

The difficult structure is

$$
\ln\left(\sum_k\cdots\right).
$$

For one Gaussian, the logarithm acts directly on the Gaussian density and produces a quadratic expression. For a mixture, the sum over components remains inside the logarithm, so the parameters of all components are coupled.

## 3.2 Why Known Labels Would Make the Problem Easy

Imagine that each hidden component label $z_n$ were known.

Then we could:

- collect all points with $z_n=1$ and fit Gaussian 1;
- collect all points with $z_n=2$ and fit Gaussian 2;
- continue independently for every component;
- set $\pi_k$ equal to the observed fraction of points in group $k$.

The only obstacle is that the labels are missing.

This motivates EM:

> estimate the missing labels probabilistically, then fit the parameters as though fractional labels were available.

## 3.3 The Circular Dependency

The learning problem contains a circle:

- To compute responsibilities, we need the component parameters.
- To estimate component parameters, we need the responsibilities.

Written schematically,

$$
\text{parameters}
\rightarrow
\text{responsibilities}
\rightarrow
\text{parameters}.
$$

EM resolves the circle through alternating updates.

## 3.4 Likelihood Singularities

Maximum likelihood for a GMM has a serious failure mode. A Gaussian component can collapse onto one data point.

> ![Figure 9.6](./CoursePR2026/Fig/Chapter_9/lecture_fig_9_6__textbook_fig_9_7__p434.png)
>
> *Figure 9.6 (Textbook Fig. 9.7, p. 434): One mixture component collapses onto a single observation. As its variance approaches zero, its density at that point becomes arbitrarily large.*

In one dimension, the Gaussian density at its own mean is

$$
\mathcal{N}(x=\mu\mid\mu,\sigma^2)
=
\frac{1}{\sqrt{2\pi}\sigma}.
$$

If

$$
\sigma\rightarrow 0,
$$

then

$$
\frac{1}{\sqrt{2\pi}\sigma}\rightarrow\infty.
$$

In a mixture, another component can continue to explain the remaining data while the collapsed component creates an extremely large density at one point. Consequently, the likelihood is unbounded above.

This is not a useful clustering solution. It is a form of overfitting.

### Practical protection

A common numerical protection is covariance regularization:

$$
\boldsymbol{\Sigma}_k
\leftarrow
\boldsymbol{\Sigma}_k+\varepsilon\mathbf{I},
$$

where $\varepsilon>0$ is small.

Other protections include detecting nearly empty components, reinitializing collapsed components, and using Bayesian or penalized methods.

## 3.5 Local Maxima

Even when singularities are avoided, the likelihood usually has multiple local maxima. EM increases the likelihood from its starting point, but does not guarantee the best possible maximum.

Therefore initialization matters.

A standard practical strategy is:

1. run K-means;
2. initialize GMM means using the K-means centres;
3. initialize covariances from the K-means clusters;
4. initialize $\pi_k$ from cluster proportions;
5. run EM;
6. repeat with several starts and keep the best valid solution.

## 3.6 Label Switching and Identifiability

If two components exchange their labels, the mixture density does not change.

For example,

$$
(\pi_1,\mu_1,\Sigma_1)
\leftrightarrow
(\pi_2,\mu_2,\Sigma_2)
$$

represents the same distribution after relabelling.

With $K$ components, there are $K!$ equivalent label permutations.

This is called a lack of identifiability of component labels. It usually does not harm clustering, but it matters when comparing parameter labels across independent runs.

# §4 The EM Algorithm for Gaussian Mixtures

> 📖 Textbook §9.2.2; selected intuition from §9.3–§9.4

## 4.1 EM in One Sentence

The expectation-maximization algorithm alternates between:

- **E-step:** estimate the hidden component membership probabilities;
- **M-step:** update the component parameters using those probabilities as weights.

The name can be remembered as

$$
\boxed{
\text{Expectation of hidden assignments}
\rightarrow
\text{Maximization over parameters}
}.
$$

For a GMM, the E-step is Bayes' theorem. The M-step is weighted Gaussian maximum likelihood.

## 4.2 Initialization

Choose initial values for

$$
\boldsymbol{\mu}_k,
\qquad
\boldsymbol{\Sigma}_k,
\qquad
\pi_k.
$$

A simple but sometimes weak choice is random initialization. A more reliable choice is to initialize means using K-means.

All mixing coefficients must satisfy

$$
\pi_k\geq 0,
\qquad
\sum_k\pi_k=1.
$$

Each covariance must be positive definite or at least numerically regularized.

## 4.3 E-step: Compute Responsibilities

Using the current parameter estimates, compute

$$
\boxed{
\gamma(z_{nk})
=
\frac{
\pi_k
\mathcal{N}(\mathbf{x}_n\mid\boldsymbol{\mu}_k,\boldsymbol{\Sigma}_k)
}{
\sum_{j=1}^{K}
\pi_j
\mathcal{N}(\mathbf{x}_n\mid\boldsymbol{\mu}_j,\boldsymbol{\Sigma}_j)
}
}.
$$

The numerator asks:

> How strongly does component $k$ explain point $n$ before normalization?

The denominator adds the explanations from all components so that the responsibilities sum to 1.

### Interpretation

If

$$
\gamma(z_{n1})=0.9,
\qquad
\gamma(z_{n2})=0.1,
$$

then point $n$ counts as 0.9 of a point for component 1 and 0.1 of a point for component 2 during the next M-step.

This fractional-count interpretation makes the M-step easy to understand.

## 4.4 Effective Cluster Size

Define

$$
\boxed{
N_k
=
\sum_{n=1}^{N}\gamma(z_{nk})
}.
$$

$N_k$ is the effective number of points assigned to component $k$.

For hard assignments, $N_k$ is an integer. For soft assignments, it need not be an integer.

Example:

$$
\gamma(z_{1k})=0.8,
\quad
\gamma(z_{2k})=0.6,
\quad
\gamma(z_{3k})=0.1.
$$

Then

$$
N_k=0.8+0.6+0.1=1.5.
$$

The component is effectively responsible for 1.5 observations.

## 4.5 M-step: Update the Means

The new mean is

$$
\boxed{
\boldsymbol{\mu}_k^{\mathrm{new}}
=
\frac{1}{N_k}
\sum_{n=1}^{N}
\gamma(z_{nk})\mathbf{x}_n
}.
$$

This is a weighted average.

Points with large responsibility pull the mean strongly. Points with small responsibility have little effect.

### Simple one-dimensional derivation

For one component, consider the weighted squared-error term

$$
E(\mu_k)
=
\frac{1}{2}
\sum_{n=1}^{N}
\gamma(z_{nk})(x_n-\mu_k)^2.
$$

Differentiate:

$$
\frac{dE}{d\mu_k}
=
\sum_{n=1}^{N}
\gamma(z_{nk})(\mu_k-x_n).
$$

Set the derivative to zero:

$$
\sum_{n=1}^{N}\gamma(z_{nk})\mu_k
=
\sum_{n=1}^{N}\gamma(z_{nk})x_n.
$$

Since $\mu_k$ is constant with respect to $n$,

$$
\mu_k
\sum_{n=1}^{N}\gamma(z_{nk})
=
\sum_{n=1}^{N}\gamma(z_{nk})x_n.
$$

Using

$$
N_k=\sum_n\gamma(z_{nk}),
$$

we obtain

$$
\mu_k
=
\frac{1}{N_k}
\sum_n\gamma(z_{nk})x_n.
$$

The vector result has exactly the same form.

## 4.6 M-step: Update the Covariances

The new covariance is

$$
\boxed{
\boldsymbol{\Sigma}_k^{\mathrm{new}}
=
\frac{1}{N_k}
\sum_{n=1}^{N}
\gamma(z_{nk})
(\mathbf{x}_n-\boldsymbol{\mu}_k^{\mathrm{new}})
(\mathbf{x}_n-\boldsymbol{\mu}_k^{\mathrm{new}})^T
}.
$$

Read the formula as a weighted covariance:

1. subtract the updated mean;
2. form an outer product;
3. weight by the responsibility;
4. add over data points;
5. divide by the effective count $N_k$.

In one dimension, this becomes a weighted variance:

$$
(\sigma_k^2)^{\mathrm{new}}
=
\frac{1}{N_k}
\sum_{n=1}^{N}
\gamma(z_{nk})
(x_n-\mu_k^{\mathrm{new}})^2.
$$

## 4.7 M-step: Update the Mixing Coefficients

The new mixing coefficient is

$$
\boxed{
\pi_k^{\mathrm{new}}
=
\frac{N_k}{N}
}.
$$

This is the effective fraction of the data explained by component $k$.

The update automatically satisfies

$$
\sum_{k=1}^{K}\pi_k^{\mathrm{new}}
=
\frac{1}{N}
\sum_{k=1}^{K}N_k.
$$

Because every point's responsibilities sum to 1,

$$
\sum_{k=1}^{K}N_k
=
\sum_{n=1}^{N}\sum_{k=1}^{K}\gamma(z_{nk})
=
\sum_{n=1}^{N}1
=N.
$$

Therefore

$$
\sum_{k=1}^{K}\pi_k^{\mathrm{new}}=1.
$$

## 4.8 The Complete GMM-EM Algorithm

Given a data set $\{\mathbf{x}_n\}_{n=1}^{N}$ and a chosen number of components $K$:

### Step 0: initialize

Choose

$$
\{\pi_k,\boldsymbol{\mu}_k,\boldsymbol{\Sigma}_k\}_{k=1}^{K}.
$$

### Step 1: E-step

For every point and component, compute

$$
\gamma(z_{nk})
=
\frac{
\pi_k\mathcal{N}(\mathbf{x}_n\mid\boldsymbol{\mu}_k,\boldsymbol{\Sigma}_k)
}{
\sum_j\pi_j\mathcal{N}(\mathbf{x}_n\mid\boldsymbol{\mu}_j,\boldsymbol{\Sigma}_j)
}.
$$

### Step 2: compute effective counts

$$
N_k=\sum_n\gamma(z_{nk}).
$$

### Step 3: M-step

$$
\boldsymbol{\mu}_k
\leftarrow
\frac{1}{N_k}\sum_n\gamma(z_{nk})\mathbf{x}_n,
$$

$$
\boldsymbol{\Sigma}_k
\leftarrow
\frac{1}{N_k}
\sum_n\gamma(z_{nk})
(\mathbf{x}_n-\boldsymbol{\mu}_k)
(\mathbf{x}_n-\boldsymbol{\mu}_k)^T,
$$

$$
\pi_k\leftarrow\frac{N_k}{N}.
$$

### Step 4: check convergence

Evaluate the log likelihood

$$
\ln p(\mathbf{X})
=
\sum_{n=1}^{N}
\ln
\left[
\sum_{k=1}^{K}
\pi_k
\mathcal{N}(\mathbf{x}_n\mid\boldsymbol{\mu}_k,\boldsymbol{\Sigma}_k)
\right].
$$

Stop when the change is sufficiently small. Otherwise return to the E-step.

## 4.9 A Small Numerical EM Iteration

Consider three one-dimensional points:

$$
x_1=0,
\qquad
x_2=2,
\qquad
x_3=5.
$$

Assume two components with equal initial mixing coefficients:

$$
\pi_1=\pi_2=0.5.
$$

To keep the arithmetic simple, suppose the current Gaussian density values are:

| Point | $\mathcal{N}_1(x_n)$ | $\mathcal{N}_2(x_n)$ |
|-------|-----------------------|-----------------------|
| 0 | 0.40 | 0.02 |
| 2 | 0.20 | 0.15 |
| 5 | 0.01 | 0.35 |

Because the mixing coefficients are equal, responsibilities are obtained by normalizing each row.

For $x_1=0$:

$$
\gamma(z_{11})
=
\frac{0.5\times0.40}{0.5\times0.40+0.5\times0.02}
=
\frac{0.40}{0.42}
\approx0.952.
$$

Therefore

$$
\gamma(z_{12})\approx0.048.
$$

For $x_2=2$:

$$
\gamma(z_{21})
=
\frac{0.20}{0.20+0.15}
\approx0.571,
$$

and

$$
\gamma(z_{22})\approx0.429.
$$

For $x_3=5$:

$$
\gamma(z_{31})
=
\frac{0.01}{0.01+0.35}
\approx0.028,
$$

and

$$
\gamma(z_{32})\approx0.972.
$$

The responsibility table is

| Point | Component 1 | Component 2 |
|-------|-------------|-------------|
| 0 | 0.952 | 0.048 |
| 2 | 0.571 | 0.429 |
| 5 | 0.028 | 0.972 |

### Effective counts

$$
N_1=0.952+0.571+0.028\approx1.552,
$$

$$
N_2=0.048+0.429+0.972\approx1.448.
$$

### Updated means

$$
\mu_1^{\mathrm{new}}
=
\frac{0.952(0)+0.571(2)+0.028(5)}{1.552}
\approx0.826.
$$

$$
\mu_2^{\mathrm{new}}
=
\frac{0.048(0)+0.429(2)+0.972(5)}{1.448}
\approx3.948.
$$

### Updated mixing coefficients

$$
\pi_1^{\mathrm{new}}
=
\frac{1.552}{3}
\approx0.517,
$$

$$
\pi_2^{\mathrm{new}}
=
\frac{1.448}{3}
\approx0.483.
$$

The first component moves toward the smaller values, while the second component moves toward the larger values. No hard decisions were required.

## 4.10 EM on the Old Faithful Data

> ![Figure 9.7](./CoursePR2026/Fig/Chapter_9/lecture_fig_9_7__textbook_fig_9_8__p437.png)
>
> *Figure 9.7 (Textbook Fig. 9.8, p. 437): EM for a two-component Gaussian mixture. Mixed colours represent uncertain responsibilities. The ellipses adapt their means, orientations, and spreads over successive iterations.*

The figure illustrates several differences from K-means.

- Assignments are soft rather than binary.
- Covariances adapt, so clusters can become elongated and rotated.
- Points near overlap regions appear purple because both components explain them.
- EM generally requires more computation and more iterations than K-means.

A practical workflow is often

$$
\text{K-means initialization}
\rightarrow
\text{GMM refinement using EM}.
$$

## 4.11 Why EM Improves the Likelihood

A full proof uses the lower-bound view in §9.4 of the textbook. For this course, the important conclusion is:

> If the E-step and M-step are performed correctly, the data log likelihood does not decrease after a complete EM cycle.

Thus

$$
\ln p(\mathbf{X}\mid\boldsymbol{\theta}^{(t+1)})
\geq
\ln p(\mathbf{X}\mid\boldsymbol{\theta}^{(t)}).
$$

However, three cautions remain:

1. increasing likelihood does not imply reaching the global maximum;
2. the algorithm may converge slowly near a solution;
3. an unregularized GMM may approach a singular solution.

## 4.12 Stopping Criteria

Common stopping criteria include:

### Log-likelihood change

Stop when

$$
\left|
\ell^{(t+1)}-\ell^{(t)}
\right|<\epsilon.
$$

A relative version is often better when likelihood magnitudes are large:

$$
\frac{|\ell^{(t+1)}-\ell^{(t)}|}{1+|\ell^{(t)}|}<\epsilon.
$$

### Parameter change

Stop when means, covariances, and mixing coefficients change very little.

### Maximum iterations

Always include a maximum iteration count to prevent accidental infinite loops.

### Numerical checks

During training, also check that:

- every covariance remains valid;
- responsibilities are finite and sum to 1;
- no component has an extremely small effective count;
- likelihood values do not become NaN or infinite.

# §5 K-means versus Gaussian Mixtures

> 📖 Textbook §9.3.2, with emphasis on the practical hard-versus-soft distinction

## 5.1 The Main Difference

K-means produces a hard assignment:

$$
r_{nk}\in\{0,1\}.
$$

A Gaussian mixture produces a soft assignment:

$$
0\leq\gamma(z_{nk})\leq1.
$$

For one point and two clusters:

| Method | Possible result |
|--------|-----------------|
| K-means | $(1,0)$ or $(0,1)$ |
| GMM | $(0.95,0.05)$, $(0.60,0.40)$, $(0.50,0.50)$, and so on |

The GMM result contains uncertainty information that K-means discards.

## 5.2 Centre versus Distribution

K-means represents each cluster only by its centre.

A GMM component represents:

- a mean;
- a covariance;
- a mixing probability.

Therefore a GMM can distinguish between clusters with different spreads and orientations.

Consider two points equally far from two means. K-means treats the distances equally. A GMM may prefer one component because that component has a larger covariance in the relevant direction or a larger mixing coefficient.

## 5.3 Objective versus Likelihood

K-means minimizes

$$
J=
\sum_{n,k}r_{nk}
\|\mathbf{x}_n-\boldsymbol{\mu}_k\|^2.
$$

A GMM maximizes

$$
\ln p(\mathbf{X})
=
\sum_n
\ln
\left[
\sum_k
\pi_k
\mathcal{N}(\mathbf{x}_n\mid\boldsymbol{\mu}_k,\boldsymbol{\Sigma}_k)
\right].
$$

The first is a geometric clustering objective. The second is a probabilistic density objective.

## 5.4 K-means as a Limiting Case of GMM

The textbook shows that K-means can be obtained from a restricted Gaussian mixture when:

- all components use the same spherical covariance;
- that covariance becomes very small;
- posterior responsibilities become almost 0 or 1.

Suppose

$$
\boldsymbol{\Sigma}_k=\varepsilon\mathbf{I}
$$

for every component. The Gaussian density contains

$$
\exp\left(
-\frac{1}{2\varepsilon}
\|\mathbf{x}-\boldsymbol{\mu}_k\|^2
\right).
$$

As

$$
\varepsilon\rightarrow0,
$$

small differences in distance are amplified strongly. Almost all posterior probability goes to the nearest mean.

Thus soft responsibilities approach hard assignments.

For this course, the conceptual result is enough:

$$
\boxed{
\text{K-means is approximately a hard, small-variance form of GMM clustering.}
}
$$

## 5.5 Comparison Table

| Property | K-means | Gaussian Mixture + EM |
|----------|---------|-----------------------|
| Assignment | Hard | Soft |
| Cluster description | Centre only | Mean, covariance, mixing weight |
| Cluster shape | Roughly spherical under Euclidean distance | Elliptical; depends on covariance choice |
| Uncertainty | No | Yes |
| Density model | No | Yes |
| Main objective | Minimize distortion | Maximize likelihood |
| Speed | Usually faster | Usually slower |
| Initialization sensitivity | Yes | Yes |
| Singular covariance issue | No | Yes |
| Good use | Fast baseline, quantization, initialization | Density estimation, overlapping clusters, probabilistic inference |

## 5.6 When Should We Use K-means?

K-means is a good choice when:

- the main goal is fast partitioning;
- clusters are compact and reasonably separated;
- uncertainty is not required;
- the data set is very large;
- a simple baseline is needed;
- GMM initialization is needed.

## 5.7 When Should We Use a GMM?

A GMM is useful when:

- clusters overlap;
- soft membership is important;
- cluster shapes differ;
- a probability density is required;
- anomaly scores or sample generation are needed;
- downstream decisions should use uncertainty.

Neither method automatically discovers meaningful semantic classes. Both optimize mathematical criteria, and their outputs must be interpreted in the context of the application.

# §6 Practical Guidance

## 6.1 A Reliable Workflow

A practical GMM workflow is:

1. clean and inspect the data;
2. standardize features when appropriate;
3. choose several candidate values of $K$;
4. initialize with K-means;
5. add covariance regularization;
6. run EM from several starting points;
7. compare final valid log likelihoods or model-selection scores;
8. inspect cluster sizes, covariance shapes, and responsibility distributions;
9. evaluate whether the result is useful for the actual engineering task.

The last step is essential. A mathematically good likelihood does not guarantee an operationally useful partition.

## 6.2 Choosing the Number of Components

There is no universally correct automatic rule for $K$.

### Domain knowledge

If the system is known to have three operating modes, starting with $K=3$ is reasonable.

### Validation likelihood

Fit the model on training data and evaluate log likelihood on held-out data. This checks density generalization.

### BIC

The Bayesian information criterion is often used:

$$
\mathrm{BIC}
=
-2\ln p(\mathbf{X}\mid\widehat{\boldsymbol{\theta}})
+M\ln N,
$$

where $M$ is the number of free parameters. Lower BIC is preferred.

The first term rewards data fit. The second penalizes complexity.

### AIC

Another criterion is

$$
\mathrm{AIC}
=
-2\ln p(\mathbf{X}\mid\widehat{\boldsymbol{\theta}})
+2M.
$$

Again, lower is preferred.

For this course, students do not need to derive AIC or BIC. The key lesson is:

> choosing a larger $K$ improves training fit, so a complexity penalty or validation set is needed.

## 6.3 Counting Parameters

Parameter count helps explain why full-covariance mixtures can overfit.

For one component in $D$ dimensions:

- mean: $D$ parameters;
- full covariance: $D(D+1)/2$ parameters;
- mixing coefficient: effectively one parameter, although one total constraint is shared across components.

For $K$ full-covariance components, the total number of free parameters is approximately

$$
K\left[D+\frac{D(D+1)}{2}\right]+(K-1).
$$

Example: $D=20$, $K=5$.

For each covariance,

$$
\frac{20\times21}{2}=210.
$$

The total is

$$
5(20+210)+4=1154.
$$

A model with more than one thousand parameters requires enough data to estimate them reliably. Diagonal covariance would need only $D=20$ variance parameters per component and would be much safer.

## 6.4 Covariance Regularization

A common implementation uses

$$
\boldsymbol{\Sigma}_k
\leftarrow
\boldsymbol{\Sigma}_k+\varepsilon\mathbf{I}.
$$

This ensures a minimum variance in every direction.

The value $\varepsilon$ should be small relative to the scaled data. This is another reason standardization helps: a single regularization scale becomes easier to choose.

## 6.5 Nearly Empty Components

A component may receive

$$
N_k\approx0.
$$

Then mean and covariance updates divide by a very small number and become unstable.

Possible responses include:

- remove the component;
- reinitialize it near a poorly explained point;
- split a large component;
- restart the entire run;
- increase regularization;
- use a smaller $K$.

Software should explicitly check for small $N_k$ rather than silently producing invalid numbers.

## 6.6 Numerical Stability in the E-step

Gaussian densities can be extremely small in high dimensions. Direct probability computation may underflow to zero.

A stable implementation works in the log domain:

$$
\log a_{nk}
=
\log\pi_k
+
\log\mathcal{N}(\mathbf{x}_n\mid\boldsymbol{\mu}_k,\boldsymbol{\Sigma}_k).
$$

Responsibilities are then normalized using the log-sum-exp identity:

$$
\log\sum_k e^{a_k}
=
m+
\log\sum_k e^{a_k-m},
$$

where

$$
m=\max_k a_k.
$$

Subtracting the maximum keeps exponentials in a numerically safe range.

Students do not need to derive this identity. They should recognize it as a standard implementation tool for probabilistic models.

## 6.7 Computational Cost

For diagonal covariances, evaluating all component densities is roughly proportional to

$$
O(NKD).
$$

For full covariances, the per-iteration cost is higher and commonly scales approximately as

$$
O(NKD^2),
$$

with additional covariance-factorization costs.

The practical message is:

- more data points increase cost linearly;
- more components increase cost linearly;
- full covariance becomes expensive as dimension grows;
- diagonal covariance is often a useful engineering compromise.

## 6.8 Pseudocode

```text
Input: data X, number of components K

Initialize means, covariances, and mixing coefficients
previous_log_likelihood = -infinity

repeat:
    # E-step
    for each point n:
        for each component k:
            score[n,k] = pi[k] * Gaussian(x[n] | mean[k], covariance[k])
        responsibility[n,:] = score[n,:] / sum(score[n,:])

    # Effective counts
    N_k = sum responsibilities over data points

    # M-step
    for each component k:
        mean[k] = weighted mean using responsibility[:,k]
        covariance[k] = weighted covariance using responsibility[:,k]
        covariance[k] += epsilon * identity
        pi[k] = N_k[k] / N

    compute current_log_likelihood

until log-likelihood change is small

Output: pi, means, covariances, responsibilities
```

## 6.9 Common Implementation Mistakes

| Mistake | Consequence | Correction |
|---------|-------------|------------|
| Responsibilities not normalized over $k$ | Rows do not sum to 1 | Normalize separately for every point |
| Updating covariance using the old mean unintentionally | Incorrect M-step | Compute new means first, then new covariances |
| Forgetting $\pi_k$ in the E-step | Ignores component frequency | Use prior times likelihood |
| No covariance regularization | Singular matrices or collapse | Add $\varepsilon\mathbf{I}$ |
| Comparing raw likelihood products | Numerical underflow | Use log likelihood |
| One random initialization only | Poor local optimum | Use multiple restarts |
| Unscaled heterogeneous features | Distances dominated by one feature | Standardize or use meaningful scaling |
| Treating component numbers as fixed semantic labels | Inconsistent labels across runs | Match components after training if needed |

## 6.10 A Practical Sanity Checklist

After fitting a GMM, verify:

$$
\sum_k\pi_k\approx1,
$$

and for every point,

$$
\sum_k\gamma(z_{nk})\approx1.
$$

Also inspect:

- all $\pi_k$ are nonnegative;
- covariance matrices are finite and positive definite;
- no component has almost zero variance;
- no component has an implausibly tiny effective count;
- the likelihood curve is nondecreasing apart from tiny floating-point noise;
- results are reasonably stable across restarts;
- cluster interpretation is consistent with domain knowledge.

# §7 The General EM Idea and Modern Connections

> 📖 Textbook §9.3–§9.4, simplified for this course

## 7.1 Complete versus Incomplete Data

Let

$$
\mathbf{X}=\text{observed data},
$$

and

$$
\mathbf{Z}=\text{latent variables}.
$$

The **complete data** would be

$$
(\mathbf{X},\mathbf{Z}).
$$

The actual **incomplete data** contain only

$$
\mathbf{X}.
$$

The observed-data likelihood sums out the hidden variables:

$$
p(\mathbf{X}\mid\boldsymbol{\theta})
=
\sum_{\mathbf{Z}}
p(\mathbf{X},\mathbf{Z}\mid\boldsymbol{\theta}).
$$

The corresponding log likelihood is

$$
\ln p(\mathbf{X}\mid\boldsymbol{\theta})
=
\ln
\sum_{\mathbf{Z}}
p(\mathbf{X},\mathbf{Z}\mid\boldsymbol{\theta}).
$$

The sum inside the logarithm is the source of difficulty.

## 7.2 The Complete-Data Graph

> ![Figure 9.8](./CoursePR2026/Fig/Chapter_9/lecture_fig_9_8__textbook_fig_9_9__p442.png)
>
> *Figure 9.8 (Textbook Fig. 9.9, p. 442): The Gaussian-mixture graphical model when both the observed data $x_n$ and component identities $z_n$ are treated as available. The plate denotes repetition over $N$ data points.*

When $z_n$ is observed, the complete-data log likelihood separates by component. This is why the M-step has simple weighted formulas.

## 7.3 General EM Template

For a model with parameters $\boldsymbol{\theta}$ and latent variables $\mathbf{Z}$:

### E-step

Use the current parameters $\boldsymbol{\theta}^{\mathrm{old}}$ to compute the posterior over hidden variables:

$$
p(\mathbf{Z}\mid\mathbf{X},\boldsymbol{\theta}^{\mathrm{old}}).
$$

Then form the expected complete-data log likelihood:

$$
Q(\boldsymbol{\theta},\boldsymbol{\theta}^{\mathrm{old}})
=
\mathbb{E}_{\mathbf{Z}\mid\mathbf{X},\boldsymbol{\theta}^{\mathrm{old}}}
\left[
\ln p(\mathbf{X},\mathbf{Z}\mid\boldsymbol{\theta})
\right].
$$

### M-step

Choose new parameters that maximize this expectation:

$$
\boldsymbol{\theta}^{\mathrm{new}}
=
\arg\max_{\boldsymbol{\theta}}
Q(\boldsymbol{\theta},\boldsymbol{\theta}^{\mathrm{old}}).
$$

For the GMM:

- the posterior over $\mathbf{Z}$ becomes responsibilities;
- maximizing $Q$ gives weighted means, covariances, and mixing coefficients.

## 7.4 The Intuition Without the Full Proof

EM can be understood as alternating between two questions.

### Question 1: What hidden explanation is plausible now?

This is the E-step.

### Question 2: What parameters best fit those probabilistic explanations?

This is the M-step.

The E-step does not permanently declare labels. It computes uncertainty under the current model. The M-step changes the model, so the next E-step may revise those beliefs.

## 7.5 EM Is Not Just for GMMs

The same pattern can appear whenever data contain missing or hidden variables.

| Model or setting | Hidden quantity | EM-style role |
|------------------|-----------------|---------------|
| Gaussian mixture | Component identity | Infer responsibilities, update Gaussian parameters |
| Hidden Markov model | Hidden state sequence | Infer state probabilities, update transition/emission parameters |
| Missing-data Gaussian model | Missing feature values | Infer missing-value statistics, update distribution parameters |
| Probabilistic PCA | Low-dimensional latent coordinates | Infer latent coordinates, update loading matrix |
| Factor analysis | Latent factors | Infer factors, update factor loadings and noise |
| Semi-supervised mixture model | Missing class labels for some samples | Infer soft labels, update class distributions |

Chapter 13 will revisit this logic for hidden Markov models.

## 7.6 Relation to Modern Machine Learning

The concepts in this chapter remain useful even when modern models do not use classical EM exactly.

### Latent variables

Modern generative models often distinguish observed variables from hidden representations. The central question remains:

$$
\text{How do we infer hidden structure from observed data?}
$$

### Alternating optimization

Many modern systems alternate between estimating hidden quantities and updating parameters. The exact mathematics may differ, but the problem-solving pattern is related.

### Variational inference

When the exact E-step posterior is too difficult to compute, we may approximate it. This leads naturally to Chapter 10.

Variational autoencoders use approximate posterior inference for latent variables, but a VAE should not be described simply as “the same as GMM-EM.” The conceptual connection is latent-variable learning and approximate inference.

### Mixture-of-experts models

A mixture-of-experts system combines several expert predictors with a gating mechanism. It shares the mixture idea, but modern MoE training is typically performed with gradient-based optimization rather than classical closed-form GMM EM.

## 7.7 What EM Does and Does Not Guarantee

EM guarantees a nondecreasing likelihood under correct exact updates.

EM does not guarantee:

- a global optimum;
- meaningful semantic clusters;
- automatic selection of $K$;
- resistance to model mismatch;
- avoidance of covariance singularities without safeguards;
- fast convergence in every problem.

A useful algorithmic guarantee should never be confused with a guarantee that the modelling assumptions are correct.

## 7.8 Bridge to Approximate Inference

In a GMM, the E-step is tractable because responsibilities have a closed-form Bayes rule.

In more complicated latent-variable models, the posterior

$$
p(\mathbf{Z}\mid\mathbf{X},\boldsymbol{\theta})
$$

may be too difficult to compute exactly.

Then the natural question is:

> What can we do when the E-step itself is intractable?

Chapter 10 develops deterministic approximations, especially variational inference and expectation propagation.

# §8 Simple Textbook Exercises and Worked Problems

## 8.1 Textbook Exercise 9.3: Recover the GMM by Marginalization

The exercise asks us to show that a latent component model produces the Gaussian-mixture density.

Suppose

$$
p(z_k=1)=\pi_k,
$$

and

$$
p(\mathbf{x}\mid z_k=1)
=
\mathcal{N}(\mathbf{x}\mid\boldsymbol{\mu}_k,\boldsymbol{\Sigma}_k).
$$

The marginal density is

$$
p(\mathbf{x})
=
\sum_{\mathbf{z}}p(\mathbf{z})p(\mathbf{x}\mid\mathbf{z}).
$$

Because the 1-of-$K$ variable has exactly one active state, the sum becomes

$$
p(\mathbf{x})
=
\sum_{k=1}^{K}
p(z_k=1)p(\mathbf{x}\mid z_k=1).
$$

Substitute the two model definitions:

$$
p(\mathbf{x})
=
\sum_{k=1}^{K}
\pi_k
\mathcal{N}(\mathbf{x}\mid\boldsymbol{\mu}_k,\boldsymbol{\Sigma}_k).
$$

This is exactly the Gaussian-mixture density.

### Lesson

Marginalization removes the hidden identity but preserves its effect as a weighted sum of components.

## 8.2 Textbook Exercise 9.1: Why K-means Terminates

K-means has a finite number of possible hard assignments. With $N$ points and $K$ clusters, there are at most

$$
K^N
$$

assignment patterns.

For any fixed assignment pattern, the optimal cluster centres are uniquely given by the means of the assigned points, except for special empty-cluster cases that must be handled separately.

Each nontrivial K-means step decreases $J$. Therefore the algorithm cannot cycle forever through distinct assignments with decreasing objective. After a finite number of changes, it reaches an assignment that no longer changes.

### Important qualification

Termination does not mean global optimality. It means only that a local stable solution has been reached.

## 8.3 Worked Problem: One K-means Iteration

Data:

$$
x=\{0,1,4,5\},
$$

with initial centres

$$
\mu_1=0,
\qquad
\mu_2=4.
$$

### Assignment

- 0 is assigned to cluster 1.
- 1 is assigned to cluster 1.
- 4 is assigned to cluster 2.
- 5 is assigned to cluster 2.

### Update

$$
\mu_1^{\mathrm{new}}
=
\frac{0+1}{2}=0.5,
$$

$$
\mu_2^{\mathrm{new}}
=
\frac{4+5}{2}=4.5.
$$

### Objective after the update

$$
J
=
(0-0.5)^2+(1-0.5)^2+(4-4.5)^2+(5-4.5)^2.
$$

Therefore

$$
J=0.25+0.25+0.25+0.25=1.
$$

## 8.4 Worked Problem: Responsibility Calculation

For a point $x$, suppose

$$
\pi_1=0.8,
\qquad
\pi_2=0.2,
$$

and

$$
\mathcal{N}_1(x)=0.1,
\qquad
\mathcal{N}_2(x)=0.4.
$$

The unnormalized scores are

$$
s_1=0.8\times0.1=0.08,
$$

$$
s_2=0.2\times0.4=0.08.
$$

Therefore

$$
\gamma(z_1)=\frac{0.08}{0.16}=0.5,
$$

$$
\gamma(z_2)=\frac{0.08}{0.16}=0.5.
$$

This example demonstrates that a strong prior for component 1 can be exactly balanced by a stronger likelihood under component 2.

## 8.5 Worked Problem: Weighted Mean Update

Suppose three points have responsibilities for component 1:

| $x_n$ | $\gamma(z_{n1})$ |
|-------|------------------|
| 1 | 0.9 |
| 3 | 0.5 |
| 8 | 0.1 |

The effective count is

$$
N_1=0.9+0.5+0.1=1.5.
$$

The updated mean is

$$
\mu_1^{\mathrm{new}}
=
\frac{0.9(1)+0.5(3)+0.1(8)}{1.5}.
$$

Compute the numerator:

$$
0.9+1.5+0.8=3.2.
$$

Thus

$$
\mu_1^{\mathrm{new}}
=
\frac{3.2}{1.5}
\approx2.13.
$$

Although $x=8$ is large, it has only 0.1 responsibility and therefore has limited influence.

## 8.6 Concept Question: Hard or Soft?

Two clusters overlap strongly. A point lies near the overlap and receives responsibilities

$$
(0.52,0.48).
$$

K-means would output one label. A GMM reports uncertainty.

Which output is more useful depends on the application.

- For a simple colour palette index, one label may be sufficient.
- For a medical or fault-detection decision, the uncertainty may be essential.
- For later probabilistic processing, retaining the full responsibility vector avoids throwing information away.

## 8.7 Concept Question: Why Can More Components Overfit?

Adding components increases flexibility. Training likelihood can generally improve because the model has more ways to fit the observations.

But excessive components may:

- model random noise;
- create tiny clusters;
- produce unstable covariance estimates;
- collapse near individual observations;
- generalize poorly to new data.

This is why $K$ should be selected using validation, information criteria, and domain judgement rather than training likelihood alone.

## 9.1 The Main Story

This chapter begins with hard clustering and gradually introduces probabilistic latent-variable learning.

The conceptual chain is

$$
\text{nearest centre}
\rightarrow
\text{hard assignment}
\rightarrow
\text{mixture component}
\rightarrow
\text{latent identity}
\rightarrow

\rightarrow
\text{EM}.
$$

## 9.2 Equations to Remember

### K-means objective

$$
J
=
\sum_{n=1}^{N}
\sum_{k=1}^{K}
r_{nk}
\|\mathbf{x}_n-\boldsymbol{\mu}_k\|^2.
$$

### K-means centre update

$$
\boldsymbol{\mu}_k
=
\frac{\sum_n r_{nk}\mathbf{x}_n}{\sum_n r_{nk}}.
$$

### Gaussian mixture density

$$
p(\mathbf{x})
=
\sum_{k=1}^{K}
\pi_k
\mathcal{N}(\mathbf{x}\mid\boldsymbol{\mu}_k,\boldsymbol{\Sigma}_k).
$$

### Responsibility

$$
\gamma(z_{nk})
=
\frac{
\pi_k\mathcal{N}(\mathbf{x}_n\mid\boldsymbol{\mu}_k,\boldsymbol{\Sigma}_k)
}{
\sum_j
\pi_j\mathcal{N}(\mathbf{x}_n\mid\boldsymbol{\mu}_j,\boldsymbol{\Sigma}_j)
}.
$$

### Effective count

$$
N_k=\sum_n\gamma(z_{nk}).
$$

### EM mean update

$$
\boldsymbol{\mu}_k
=
\frac{1}{N_k}
\sum_n\gamma(z_{nk})\mathbf{x}_n.
$$

### EM covariance update

$$
\boldsymbol{\Sigma}_k
=
\frac{1}{N_k}
\sum_n\gamma(z_{nk})
(\mathbf{x}_n-\boldsymbol{\mu}_k)
(\mathbf{x}_n-\boldsymbol{\mu}_k)^T.
$$

### EM mixing-weight update

$$
\pi_k=\frac{N_k}{N}.
$$

## 9.3 What Students Should Be Able to Explain in Words

A student should be able to say:

> K-means repeatedly assigns every point to the nearest centre and then replaces each centre by the mean of its assigned points.

A student should also be able to say:

> A Gaussian mixture assumes every point was generated by one hidden Gaussian component. EM first computes the posterior probability of each component for every point, then updates the Gaussian parameters using those probabilities as fractional weights.

These verbal explanations are at least as important as memorizing equations.

## 9.4 Common Student Confusions

| Confusion | Clarification |
|-----------|---------------|
| “K-means estimates Gaussian covariances.” | It does not. It estimates only centres and hard assignments. |
| “A responsibility is a distance.” | It is a posterior probability obtained from prior times likelihood. |
| “The nearest mean always has the highest GMM responsibility.” | Not necessarily; covariance and mixing weight also matter. |
| “EM directly observes the hidden labels.” | No. It computes their posterior probabilities. |
| “EM always finds the best solution.” | It usually finds a local maximum determined partly by initialization. |
| “Likelihood increasing means the clusters are meaningful.” | No. It only means the chosen probabilistic objective improved. |
| “The number of components equals the number of real classes.” | Not necessarily. Components are statistical building blocks. |
| “A larger $K$ is always better because likelihood improves.” | Larger models can overfit and require model selection. |
| “K-means and GMM are unrelated algorithms.” | K-means is closely related to a hard, small-variance limit of a GMM. |

## 9.9 Final Takeaway

The most important lesson is not a long derivation. It is the reusable idea:

$$
\boxed{
\text{When hidden information makes learning difficult,}
\quad
\text{infer it softly and update the model alternately.}
}
$$

K-means gives the simplest hard-assignment version of this idea. Gaussian-mixture EM gives a probabilistic soft-assignment version. Chapter 10 asks how to continue when exact posterior inference is no longer easy.
